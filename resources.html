Compression

Windows executable packers

There are two main alternatives to compress native binaries on Windows: kkrunchy and squishy.

kkrunchy

kkrunchy was written by Farbrausch and has been the most popular executable compression tool for 64k intros since its release in 2006. It is simple to use, fast and efficient. The fork rekkrunchy provides additional information to help analyse the file size. Note that the generated binaries are generally flagged as malware by antivirus vendors, and only 32-bit Windows binaries are supported.

  • kkrunchy project page and source code
  • rekkrunchy project page and source code
  • Article: x86 code compression in kkrunchy

squishy

squishy is a recent and modern alternative to kkrunchy, written by Logicoma. It generally gets better results than kkrunchy in terms of final size (see the metrics), but is also significantly slower, so it offers a different tradeoff. Both Windows 32-bit and 64-bit binaries are supported.

  • squishy project page
  • squishy / kkrunchy comparison
  • Revision 2020 seminar: modern 64k intro compression
  • Blog post with some details about squishy

Crinkler

While Crinkler is the main executable compression tool for 4k and 8k intros, its use for sizes like 32k and 64k is not recommended.

Shader minification

Shader Minifier

Shader Minifier written by Ctrl-Alt-Test and is the most common shader minification tool for GLSL and HLSL. It rewrites shader source code in a way that uses a smaller footprint and compresses well, and options are exposed to choose the transformations to apply.

  • Shader Minifier project page and source code
  • Revision 2023 seminar: Shader minification for 4k - 64k intros

Music
